# PRD: dedaloproteccion.mx
# Sitio web de asesoría de seguros para José G. Helú
# Marca: DÉDALO · Arquitectura de Protección

## OBJETIVO GENERAL
Sitio production-ready con UX completa, contenido real y todas las integraciones funcionando para el negocio de asesoría de seguros de José G. Helú bajo la marca DÉDALO · Arquitectura de Protección.

## STACK TECNOLÓGICO
- Framework: Next.js 16 con App Router
- Estilos: Tailwind CSS 4
- Lenguaje: TypeScript
- Deploy: Vercel
- Dominio: dedaloproteccion.mx

### Objetivo
Consistencia visual y de experiencia en todas las páginas del sitio.

### Requerimientos
1.1. Revisión y corrección de consistencia de marca (colores, tipografías, espaciados) en todas las páginas.

1.7. Loading states (skeletons o spinners) en componentes que hagan fetch de datos.
1.8. Footer completo con links, datos de contacto y WhatsApp.

## MÓDULO 3: DIAGNÓSTICO (/diagnostico)

### Objetivo
Herramienta interactiva que evalúa las necesidades de seguros del visitante y genera un perfil personalizado.

### Requerimientos
3.1. Componente DiagnosticoForm completamente funcional con flujo multi-paso:
  - Sección 1: Datos personales (nombre, edad, estado civil, dependientes)
  - Sección 2: Situación laboral (empleado, independiente, empresario)
  - Sección 3: Patrimonio y activos (vehículos, inmuebles)
  - Sección 4: Salud y preexistencias
  - Sección 5: Objetivos de protección
3.2. Navegación entre secciones con botones "Siguiente" y "Atrás", barra de progreso visual.
3.3. Validación de campos requeridos antes de avanzar.
3.4. Lógica de scoring: algoritmo que asigna puntos por respuesta y determina un perfil de protección.
3.5. Perfiles posibles: Básico, Esencial, Completo, Premium (4 perfiles).
3.6. Página de resultado personalizada por perfil mostrando:
  - Nombre del perfil
  - Descripción de necesidades identificadas
  - Seguros recomendados
  - Botón "Agendar cita con José" → /agenda
3.7. Webhook POST a Make/Integromat para enviar datos del diagnóstico a Notion.
  - URL del webhook en variable de entorno: NEXT_PUBLIC_MAKE_WEBHOOK_URL
  - Datos a enviar: nombre, email, teléfono, perfil asignado, respuestas clave, timestamp
3.8. Variables de entorno configuradas en Vercel para producción.
3.9. Estado de loading durante envío del webhook, confirmación o manejo de error.
